# OneStep 问题优先级列表

## 🔴 高优先级（影响功能、安全或核心质量）

### 1. WebHookBroker 安全问题 ⚠️
- **问题**: 默认监听 `0.0.0.0`，任何人都可以访问；没有认证机制
- **影响**: 严重的生产安全风险，可能导致未授权访问和消息注入
- **建议**: 添加 API key 或签名验证；默认监听 `127.0.0.1`
- **优先级**: P0（最高）
- **状态**: ✅ 已完成

### 2. 线程管理问题
- **问题**: `ThreadPoolWorker.shutdown()` 只调用 `executor.shutdown()`，没有等待线程池完全关闭
- **影响**: 可能导致优雅关闭失败，资源未正确释放
- **建议**: 添加 `wait=True` 参数，确保线程池完全关闭
- **优先级**: P0
- **状态**: ✅ 已完成

### 3. 内存泄漏风险
- **问题**: `WebHookBroker._servers` 和 `WebHookServer.servers` 是全局字典，没有清理机制
- **影响**: 长期运行会累积数据，内存持续增长
- **建议**: 添加清理机制，定期清理过期或无用的服务器实例
- **优先级**: P0
- **状态**: ✅ 已完成

### 4. 测试覆盖不全面
- **问题**: 缺少集成测试和压力测试；有些测试依赖外部服务（如 RabbitMQ）
- **影响**: 无法保证代码在生产环境的可靠性
- **建议**: 添加集成测试套件；使用 mock 隔离外部依赖
- **优先级**: P1

### 5. 配置管理
- **问题**: 缺少配置文件支持（如 `config.yaml` 或 `.onestep.yml`）
- **影响**: 所有参数都要硬编码在代码里，不利于部署和管理
- **建议**: 添加配置文件支持，支持环境变量覆盖
- **优先级**: P1
- **状态**: ✅ 已完成

### 6. 日志敏感信息
- **问题**: `message.__str__` 会输出完整的 body，可能包含敏感信息
- **影响**: 可能导致敏感数据泄露
- **建议**: 添加日志脱敏配置；支持自定义消息序列化
- **优先级**: P1
- **状态**: ✅ 已完成

## 🟡 中优先级（影响开发体验或可维护性）

### 7. 文档字符串不完整
- **问题**: 很多关键类和方法没有 docstring，特别是 `broker` 和 `middleware` 模块
- **影响**: 代码可读性和可维护性差
- **建议**: 补充公共 API 的文档字符串
- **优先级**: P2

### 8. README 示例不够
- **问题**: 只展示了最基础的用法；缺少实际场景示例
- **影响**: 新手难以快速上手
- **建议**: 添加更多示例（多 broker 组合、中间件使用、错误处理）
- **优先级**: P2

### 9. 中间件链
- **问题**: 中间件的 before/after 钩子没有优先级控制
- **影响**: 无法灵活控制中间件执行顺序
- **建议**: 添加 `order` 参数或支持链式调用
- **优先级**: P2

### 10. 信号系统
- **问题**: 使用 `blinker` 库，但文档里没说明有哪些信号
- **影响**: 用户不知道可以监听哪些事件
- **建议**: 在 README 或文档中列出所有可用信号
- **优先级**: P2

### 11. 消息队列阻塞
- **问题**: `MemoryBroker` 的 `maxsize` 默认为 0（无限制），可能导致内存溢出
- **影响**: 生产环境可能导致 OOM
- **建议**: 设置合理的默认值或提供文档说明
- **优先级**: P2

### 12. 版本管理
- **问题**: `__version__` 定义在 `__init__.py` 中，但 `pyproject.toml` 也要手动同步
- **影响**: 容易导致版本号不一致
- **建议**: 只在一个地方维护版本号（如使用 `__version__` 动态读取）
- **优先级**: P2

### 13. 依赖管理
- **问题**: `dev` 和 `test` 依赖重复了（`pytest`、`redis` 等）
- **影响**: 不够清晰，可能导致重复安装
- **建议**: 合并或使用更清晰的分组
- **优先级**: P2

## 🟢 低优先级（锦上添花）

- **注意**: 已部分改进（BaseConsumer, MemoryBroker），但仍需检查其他地方
- **优先级**: P3

- **问题**: `BaseBroker.send()` 已有重试逻辑，但 `publish()` 方法也可能失败
- **影响**: 可能丢失消息
- **优先级**: P3

- **问题**: `.github/workflows/test.yml` 需要添加代码覆盖率、多 Python 版本测试
- **影响**: 无法自动检测回归问题
- **优先级**: P3

### 19. CLI 体验改进
- **问题**: 错误信息不够友好，缺少 `--version` 参数
- **注意**: ✅ 已完成
- **优先级**: P3

### 20. 最佳实践文档
- **问题**: 缺少性能对比、最佳实践等文档

- **优先级**: P4
